# Release Smoke Checklist - 2026-05-23

Use this checklist against a clean preview build and a seeded local database. Mark each item pass/fail with notes and attach screenshots or logs for failures.

## Inspector Flows

- [ ] Task row opens the global Inspector; dirty Escape keeps it open; Save/Delete route through preload.
- [ ] Calendar event create/edit opens the global Inspector; dirty Escape keeps it open; Save/Delete route through preload.
- [ ] Note row opens the global Inspector; edit debounce saves; close flushes the latest draft.
- [ ] Settings diagnostics details open the Inspector with sanitized JSON; token-like content is redacted.
- [ ] Platform capability row opens the Inspector with state, remediation, severity, and metadata.

## Task And Today

- [ ] Quick-add parses planned time, duration, locked schedule, and tags.
- [ ] Task perspectives switch between Inbox, Forecast, Review, Tags, Projects, and saved views with visible focus state.
- [ ] Today timeline shows all-day, morning, afternoon, evening, and unscheduled groups.
- [ ] Today conflict markers and over-capacity state are visible with the local schedule suggestion fixture.

## Notes

- [ ] Note link autocomplete suggests local notes, tasks, and events without Google calls.
- [ ] Broken links are visible from the note inspector and can be repaired through the autocomplete flow.
- [ ] Link chips are keyboard activatable.

## Calendar

- [ ] Quick-create from day, week, and month cells opens the event Inspector.
- [ ] Recurrence edit persists daily/weekly/monthly/yearly rules and shows the Repeat summary.
- [ ] Timezone display is unambiguous for calendar lists, event rows, and Settings default timezone.
- [ ] Drag/drop move and resize call `calendar.update` directly and do not open the Inspector.
- [ ] Keyboard move from a focused day event uses Arrow Up/Down in 15-minute increments.
- [ ] Pending/failed event mutation badges appear on event rows and in the Inspector.

## Platform

- [ ] Tray icon, global shortcut, OAuth loopback, credential storage, notifications, protocol registration, and MCP loopback rows show accurate state/remediation.
- [ ] Notification permission request updates Settings/Notifications state.
- [ ] MCP disabled/enabled state is reflected without exposing bearer tokens.

## Gate

- [ ] `pnpm typecheck`
- [ ] `pnpm test:unit`
