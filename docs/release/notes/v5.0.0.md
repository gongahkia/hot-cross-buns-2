# Hot Cross Buns 2 5.0.0

Unsigned macOS preview release with Linux AppImage and Windows NSIS technical
preview artifacts prepared for review. Linux/Windows automated preview gates
passed on 2026-06-14; target desktop manual QA remains required before
publishing those platform artifacts.

## Highlights

- Declares Hot Cross Buns 2 as the maintained release line.
- Ships the Electron-first macOS client for Google Tasks and Google Calendar.
- Adds account workspaces, command palette flows, menu bar surfaces, templates, reminders, smart rescheduling, attachments, ICS import/subscriptions, portable exports, MCP/CLI/webhook automation, and local customization.
- Includes image-derived custom themes and Settings controls for custom backgrounds.
- Publishes stable `Hot-Cross-Buns-2-macOS.dmg` and `Hot-Cross-Buns-2-macOS.zip` aliases plus SHA-256 checksums.
- Prepares Linux AppImage preview artifacts with stable
  `Hot-Cross-Buns-2-linux.AppImage` and
  `Hot-Cross-Buns-2-linux-x64.AppImage` aliases plus SHA-256 checksums. Publish
  after the Ubuntu GNOME manual matrix passes.
- Prepares Windows NSIS preview artifacts with stable
  `Hot-Cross-Buns-2-windows.exe` and
  `Hot-Cross-Buns-2-windows-x64.exe` aliases plus SHA-256 checksums. Publish
  after Windows 11 installed-app QA passes.

## Install

### macOS

1. Download `Hot-Cross-Buns-2-macOS.dmg`.
2. Download `SHASUMS256.txt`.
3. Verify checksums:

```sh
shasum -a 256 -c SHASUMS256.txt
```

4. Open the DMG and move `Hot Cross Buns 2.app` into `Applications`.
5. If macOS blocks first launch, use `System Settings > Privacy & Security > Open Anyway`.

### Linux AppImage

Publish this artifact only after the Ubuntu LTS GNOME manual matrix passes.

1. Download `Hot-Cross-Buns-2-5.0.0-linux-x86_64.AppImage`.
2. Download `SHASUMS256.txt`.
3. Verify checksums:

```sh
sha256sum -c SHASUMS256.txt
```

4. Make the AppImage executable:

```sh
chmod +x Hot-Cross-Buns-2-5.0.0-linux-x86_64.AppImage
```

5. Launch it:

```sh
./Hot-Cross-Buns-2-5.0.0-linux-x86_64.AppImage
```

### Windows NSIS

Publish this artifact only after Windows 11 x64 packaging, install, runtime,
and uninstall QA pass.

1. Download `Hot-Cross-Buns-2-5.0.0-windows-x64.exe`.
2. Download `SHASUMS256.txt`.
3. Verify checksums with PowerShell:

```powershell
Get-FileHash .\Hot-Cross-Buns-2-5.0.0-windows-x64.exe -Algorithm SHA256
```

4. Install with the NSIS installer.

Windows preview support and retained-data policy are documented in
`docs/support/windows-preview-support.md`.

## Known Issues

- This preview is unsigned and unnotarized.
- In-place auto-update is intentionally disabled; Settings > About checks GitHub Releases and opens a manual download path.
- Google sync uses your own Google Cloud Desktop OAuth client.
- Linux is a technical preview scoped first to Ubuntu LTS GNOME.
- Linux local notifications, global shortcuts, tray/status area, open-at-login,
  and `hotcrossbuns://` deep links are explicitly unsupported in this preview.
- Linux credential storage requires an OS-backed provider such as GNOME
  Keyring/libsecret or KWallet; plaintext fallback is refused.
- Windows is not public-support ready until NSIS install, Start Menu identity,
  taskbar grouping, notifications, deep links, autostart, OAuth, MCP, SQLite,
  uninstall, and SmartScreen behavior pass Windows 11 QA.
- Windows preview installers are unsigned unless a signing plan is configured;
  public Windows distribution requires Authenticode signing and SmartScreen
  expectations. Expected unsigned-preview behavior is documented in
  `docs/release/windows-signing-smartscreen.md`.
